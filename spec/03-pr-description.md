# 03 — PR Description

**Version:** 1.0.0-draft  
**Status:** Candidate  
**Schema:** [`schemas/pr-description.json`](../schemas/pr-description.json)

## Overview

AI generates PR descriptions automatically — but are they complete? Do they disclose AI involvement? Do they link to issues? ODS defines a structured PR description format that tools can validate, and reviewers can rely on.

## Specification

### PR Description Template

```markdown
## Summary
[One sentence describing what this PR does]

## Type
- [ ] Feature
- [ ] Bugfix
- [ ] Hotfix
- [ ] Refactor
- [ ] Documentation
- [ ] Chore

## AI Disclosure
- [ ] This PR contains AI-generated code
- **AI Tool:** [e.g., GitHub Copilot, Claude, Cursor]
- **AI Scope:** [What the AI generated]
- **Human Review:** [What the human reviewed/modified]

## Related Issues
Closes #[issue-number]
Related to #[issue-number]

## Changes
- [change 1]
- [change 2]

## Testing
- [ ] Unit tests added/updated
- [ ] Integration tests added/updated
- [ ] Manual testing performed
- **Test coverage before:** XX%
- **Test coverage after:** XX%

## Risk Assessment
- **Deployment risk:** [Low / Medium / High]
- **Rollback plan:** [Link or brief description]
- **Breaking change:** [Yes / No]

## Checklist
- [ ] Branch naming follows [ODS Branch Naming](01-branch-naming.md)
- [ ] Commits follow [ODS Commit Message](02-commit-message.md)
- [ ] AI-generated code has been reviewed by a human
- [ ] No secrets, tokens, or credentials are included
- [ ] Documentation has been updated

## Screenshots / Evidence
[Optional: screenshots, logs, test results]
```

### Required Fields

| Field | Required | Description |
|-------|----------|-------------|
| `summary` | **Yes** | One-sentence description of the change |
| `type` | **Yes** | At least one checkbox must be selected |
| `ai_disclosure` | **Yes** | Must declare whether AI was used |
| `changes` | **Yes** | At least one line describing changes |
| `testing` | **Yes** | At least `Manual testing performed` checked |
| `checklist` | **Yes** | All items must have a response |

### AI Disclosure Rules

1. If ANY code was generated by AI → `This PR contains AI-generated code` must be checked
2. If checked, `AI Tool` and `AI Scope` become required
3. `Human Review` must describe what a human verified — not just "Reviewed"

### Examples

**AI-generated feature PR:**
```markdown
## Summary
Add OAuth 2.0 login flow supporting Google, GitHub, and Microsoft providers.

## Type
- [x] Feature
- [ ] Bugfix
- [ ] Hotfix
- [ ] Refactor
- [ ] Documentation
- [ ] Chore

## AI Disclosure
- [x] This PR contains AI-generated code
- **AI Tool:** GitHub Copilot + Claude
- **AI Scope:** Provider configuration, token exchange, session management
- **Human Review:** Verified OAuth spec compliance, added PKCE, fixed redirect URI validation

## Related Issues
Closes #42

## Changes
- Added OAuth provider abstraction layer
- Implemented Google, GitHub, Microsoft provider configs
- Added PKCE support for all flows
- Created session management with refresh token rotation

## Testing
- [x] Unit tests added/updated
- [x] Integration tests added/updated
- [x] Manual testing performed
- **Test coverage before:** 72%
- **Test coverage after:** 78%

## Risk Assessment
- **Deployment risk:** Medium
- **Rollback plan:** Feature flag `oauth-v2` controls rollout. Disable flag to rollback.
- **Breaking change:** No

## Checklist
- [x] Branch naming follows ODS
- [x] Commits follow ODS
- [x] AI-generated code has been reviewed by a human
- [x] No secrets, tokens, or credentials are included
- [x] Documentation has been updated
```

## JSON Schema Validation

The [pr-description.json schema](../schemas/pr-description.json) validates:
1. Required sections exist
2. AI disclosure is properly filled when AI was used
3. Type is selected (at least one)
4. Risk assessment includes rollback plan for Medium/High risk changes

## Tooling

### CLI
```bash
# Validate a PR description
ods validate pr --file PR_BODY.md

# Generate a PR template
ods generate pr --type feature --ai-tool "GitHub Copilot"
```

### GitHub Action
```yaml
- uses: open-delivery-spec/github-action@v1
  with:
    check: pr-description
    pr-body: ${{ github.event.pull_request.body }}
```
